Project Brief: Kinetic Workout Execution Screen Redesign
1. Project Overview
Product Name: Kinetic (Fitness Training App)

Project Owner: Product Manager

Objective: Redesign the primary "Workout Execution" screen to improve readability and usability during training sessions. The goal is to provide a clean, card-based interface that clearly displays daily exercises and their specific metrics while maintaining the established brand identity and color palette.

2. Target Audience
Active Gym-Goers: Users who need a quick, glanceable reference for their sets, reps, and weights while in the middle of a workout.

The "Power User": Users who frequently switch between light and dark environments (e.g., outdoor vs. indoor gym) and rely on the theme toggle.

3. Functional Requirements
3.1 Global Navigation & Header
Brand Header: The "KINETIC" logo must remain fixed at the top of the screen.

Theme Toggle: A toggle switch (Sun/Moon icon) must be located in the top-right corner to allow instantaneous switching between Light and Dark modes.

Bottom Navigation: The existing 4-tab bar (TRAIN, STATS, SOCIAL, GEAR) must remain intact and accessible at all times.

3.2 Daily Exercise List
Exercise Cards: The main content area will display a vertical list of exercises scheduled for the day (e.g., Supino Reto, Desenvolvimento Livre, Elevação Lateral).

Required Data Points per Exercise: Each card must display the Exercise Name (clear, bold typography), Sets & Repetitions (e.g., 3x 10-12), Suggested Weight (e.g., 80kg), and Rest Time (e.g., 60s).

4. UI/UX Design Requirements
4.1 Visual Identity
Color Palette: Strictly adhere to the existing Kinetic palette.

Light Mode: Soft grey background with high-contrast white cards.

Dark Mode: Deep dark grey background (#131313) with neon blue accents for highlights/active states.

Layout: Clean, minimalist card design to reduce cognitive load during high-intensity training.

4.2 Theme Variations
Light Mode: Focus on readability under bright lights.

Dark Mode: Focus on sleekness and reduced eye strain, utilizing neon blue for branding and key action elements.

5. User Stories
As a user, I want to see my entire workout at a glance so I can plan my movement through the gym.

As a user, I want to know exactly how long to rest and what weight to use without having to click through multiple menus.

As a user, I want to switch to dark mode easily when training in low-light environments.

6. Future Considerations (Phase 2)
The following features were identified as high-value additions for the next iteration:

Integrated Rest Timer: An active countdown timer that triggers when a set is marked as complete.

Exercise Library Integration: Clickable cards that lead to a detail page with a video demonstration of the exercise.

Active Logging: An input field within each card to allow users to record the actual weight lifted versus the suggested weight.

7. Success Metrics
User Retention: Increase in daily active users (DAU) returning to the "Train" tab.

Engagement: Reduction in time spent on the screen (indicating users are finding the info they need quickly).

UX Feedback: Positive sentiment regarding the visual consistency between light and dark modes.
